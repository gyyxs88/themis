Decision 最小对象首版

先把老板拍板、取舍原因和影响范围收成结构化记录，并且先挂回 `asset / work item`，避免关键结论继续只散在聊天里。

当前筛选下还没有决策记录。

新建决策

最小字段：title / status / decidedAt / related ids

这一版先直接记 ids，不在这里做复杂关系图；先把老板拍板从聊天里拔出来。

风险 / 事故

Risk / Incident 最小对象首版

先把异常、故障、权限风险和合规风险升级成可跟踪对象，并先挂回 `asset / decision / work item`，避免高风险事项继续只留在任务描述里。

当前筛选下还没有风险记录。

新建风险

最小字段：type / severity / status / related ids

这一版先直接记 `risk | incident`、严重度和关联 ids，先把高风险事项从聊天里单独拔出来。

节奏记录

Cadence 最小对象首版

先把周检、续费、备份抽查和账单复盘这类固定动作收成长期节奏，并先挂回 `asset / playbook`，避免它们继续只靠单次 scheduled task 勉强表达。

当前筛选下还没有节奏记录。

新建节奏

最小字段：title / frequency / status / nextRunAt

这一版先把重复动作收成一等对象，不急着在这里做自动触发编排；先把节奏台账立住。

承诺目标

Commitment 公司层对象首版

先把季度主线、阶段承诺和必须完成的公司级目标收成对象，并挂回 `asset / decision / risk / cadence / work item`，让老板视图能看到“哪些承诺正在冒烟”，也能继续追它兑现到了哪一步。

当前筛选下还没有承诺目标。

新建承诺

字段：title / status / progress / milestone / evidence / related ids

这一版仍不做完整 OKR；先让承诺能看到进度、关键里程碑、证据，以及来自关系边的入/出反链。

关系边

OperationEdge 最小对象首版

先把 `asset / commitment / decision / risk / cadence / work item` 之间的关系收成可查询边；当前 `Decision / Risk / Cadence / Commitment` 保存时会自动补基础关系边，手工边仍可在这里维护。

当前筛选下还没有关系边。

新建关系边

最小字段：from -> relation -> to

自动边来自对象字段，过期后会归档；这里仍保留手工关系维护入口，用来补充自动规则覆盖不到的事实。

对象详情 / 图查询
